# 拉野计算规则 v1

本文档定义当前“拉野”事件的自动候选计算口径。拉野成立本身以“某一路线兵与真正中立野怪发生交战”为核心证据；执行者归因优先使用英雄/召唤物与该组被拉野野怪的交互记录，找不到交互者时再用位置做兜底。

## 1. 事件定义

在前期对线阶段，某一方边路线上小兵与真正中立野怪发生连续伤害、死亡交互，或持续贴近且双方出现血量变化，说明该方该路兵线被野怪牵引，记为该方该路发生“拉野”。

示例：

- 天辉下路小兵和中立野怪打起来：记为“天辉下路拉野”。
- 夜魇上路小兵和中立野怪打起来：记为“夜魇上路拉野”。

这里的“天辉/夜魇哪一路”由线兵阵营和线兵所属路线决定，不直接由英雄决定。

## 2. 数据来源

主要数据源：

- `combat_logs`
  - `time`
  - `log_index`
  - `type`
  - `attackername`
  - `targetname`
  - `sourcename`
  - `targetsourcename`
  - `value`
  - `x`
  - `y`

- `dota_model_neutral_siege_creep`
  - `type`
  - `unit`
  - `team`
  - `ehandle`
  - `x`
  - `y`
  - `hp`
  - `lifeState`
  - `ownerClass`
  - 如果某场比赛缺少 `ehandle`，仍使用同秒状态点做坐标匹配和贴近判断；但不把无 `ehandle` 的单点状态当作出生路线轨迹。

辅助数据源：

- `player_intervals2`
  - 用于查找拉野地点附近英雄。
  - 当 combat_logs 没有英雄/召唤物与该组野怪的交互证据时，用作执行者兜底。

## 3. 基础筛选

时间范围：

- 默认只看前 15 分钟：`0 <= time <= 900`。

路线范围：

- 只看边路：
  - 上路 `top`
  - 下路 `bot`
- 中路 `mid` 不输出拉野。

线兵定义：

- 天辉线兵：
  - `npc_dota_creep_goodguys_melee`
  - `npc_dota_creep_goodguys_ranged`
  - `npc_dota_creep_goodguys_flagbearer`
  - `npc_dota_goodguys_siege`

- 夜魇线兵：
  - `npc_dota_creep_badguys_melee`
  - `npc_dota_creep_badguys_ranged`
  - `npc_dota_creep_badguys_flagbearer`
  - `npc_dota_badguys_siege`

真正中立野怪定义：

- 单位名是 `npc_dota_neutral_*`。
- 并且在 `dota_model_neutral_siege_creep` 中满足：
  - `type = neutral_status`
  - `team = 4`
  - `ownerClass` 为空，或不是英雄控制来源。

这个过滤用于排除 Lycan、Chen、Enchantress、支配头盔等控制的“中立单位”。

## 4. 单条证据规则

一条 `combat_logs` 记录进入拉野候选证据，需要满足：

- `type` 是：
  - `DOTA_COMBATLOG_DAMAGE`
  - 或 `DOTA_COMBATLOG_DEATH`
- 交互双方一边是线上小兵，一边是真正中立野怪。
- 方向不限：
  - 线兵打野怪
  - 野怪打线兵
  - 线兵击杀野怪
  - 野怪击杀线兵
- 排除英雄、召唤物、幻象、英雄控制野怪和普通野区打野。

如果 `combat_logs` 只有死亡记录、缺少连续伤害记录，则以该条线兵-野怪直接交互作为锚点，再读取锚点前 `45s` 到后 `5s` 的 `dota_model_neutral_siege_creep` 状态：

- 同一线兵阵营。
- 同一路线。
- 同名中立野怪。
- 线兵与野怪距离 `<= 550` 码。
- 实现时用 raw 坐标计算距离后换算成游戏码，`550` 码约等于 `4.23 raw`。

这些状态贴近记录也计入同一拉野组的交互证据。

如果 `combat_logs` 完全没有留下线兵-野怪直接交互锚点，也会全局扫描 `dota_model_neutral_siege_creep` 的状态点。只要边路线兵与真正中立野怪距离 `<= 550` 码，并且形成连续片段，再用连续秒数和线兵/野怪掉血门槛确认是否输出。这样可以覆盖日志不记录互打、但状态表能证明双方已经交战的情况。

只有状态贴近、完全没有 combat 直接交互锚点时，必须额外满足连续秒数和线兵/野怪掉血门槛；否则不输出拉野，避免把线兵路过野点误判为拉野。

## 5. 路线判断

路线判断优先级：

1. 优先用 `dota_model_neutral_siege_creep` 中线兵 `ehandle` 的全段主路线判断路线。
   - 同一只线兵在前期多数时间所在的路线，即为该线兵所属路线。
   - 这是为了避免小兵出生/出门早期坐标落在中路判定区，导致边路拉野被误判为中路。
2. 如果全段主路线缺失，再使用出生/早期轨迹判断来源路线。
3. 如果无法匹配 `ehandle`，使用交互点、死亡点或状态贴近点坐标兜底：
   - `y - x > 35`：上路
   - `x - y > 35`：下路
   - 其他：中路
3. 判断为中路时，不输出拉野。

## 6. 阵营判断

阵营由线兵决定：

- `goodguys` 线兵：天辉该路拉野。
- `badguys` 线兵：夜魇该路拉野。

注意：这里表示“该方该路线兵被拉野”。执行者需要额外看英雄/召唤物与该组野怪的交互，或位置兜底。

## 7. 聚合规则

候选证据按以下维度聚合：

- 同一线兵阵营
- 同一路线
- 相近时间
- 相近位置/野区区域

当前参数：

- 时间间隔：`<= 35s` 归为同一组。
- 位置距离：`<= 28 raw` 归为同一组。
- 如果 `combat_logs` 缺少坐标，则用线兵/野怪状态表在相近时间、相近位置的匹配结果补坐标。
- 如果 `combat_logs` 缺少连续伤害，只给出单个死亡锚点，则用锚点前 `45s` 到后 `5s` 的状态贴近记录补足连续交战证据。
- 如果状态贴近组没有 combat 锚点，则要求至少 `6` 个有效秒点，并且线兵与野怪各自至少出现 `20` 点血量下降，才允许输出。

输出门槛：

- 同组交互记录数 `>= 3`，或
- 同组线兵/野怪死亡记录数 `>= 2`。

满足任一门槛，输出一条“拉野”事件。

## 8. 英雄归因

拉野成立以后，再判断是谁拉的。

优先使用 `combat_logs` 归因：

- 只看拉野阵营同侧英雄。
- 只看该拉野组里被状态/交互证据命中的中立野怪。
- 时间窗口：拉野开始前 `30s` 到开始后 `10s`。
- 如果英雄本体攻击该组野怪，记为该英雄执行。
- 如果该组野怪攻击英雄本体，记为该英雄执行。
- 如果召唤物、支配单位、树人、小狼等与该组野怪交互，且 `sourcename` 或 `targetsourcename` 能映射到英雄，则记为所属英雄执行。

如果没有交互证据，再使用 `player_intervals2` 位置兜底：

- 时间窗口：拉野开始前 `30s` 到开始后 `10s`。
- 只看拉野阵营同侧英雄。
- 执行者距离拉野中心 `<= 800` 码，取最近者。

附近英雄仍会用 `player_intervals2` 在拉野中心 `1500` 码内、事件前后 `8s` 内补充，作为环境证据写入结果列，但不再放入 `heroes` 主列。

输出解释：

- `heroes` 字段记录推断出的拉野执行者；如果没有英雄/召唤物交互证据，也没有 800 码内同侧英雄，则写未知。
- `结果` 字段明确写“某方某路小兵与中立野怪发生连续交战，判定为某方某路拉野”。
- `结果` 字段继续保留附近英雄，方便复核当时环境。
- `evidence` 字段写 `pull_executor method=neutral_interaction` 或 `method=position_nearest`。

## 9. 输出格式

标签：

- `拉野`

置信度：

- 当前写为“未知”。
- 后续通过人工批准样本做假设检验，验证该口径的 precision/recall 后再确定模型置信度。

结果示例：

```text
天辉下路小兵与中立野怪发生连续交战，判定为天辉下路拉野；交互数 2，线兵死亡 1，野怪死亡 1；附近天辉英雄：先知、树精卫士；附近夜魇英雄：森海飞霞
```

证据示例：

```text
pull_status_proximity anchor_log_index=90385 time=87 lane_unit=npc_dota_creep_goodguys_ranged neutral_unit=npc_dota_neutral_kobold lane=bot xy=(163.7,82.9) distance_raw=2.8 lane_status_ehandle=2345468 neutral_status_ehandle=9111962
```

## 10. 需要规避的问题

1. 控制野怪误判
   - Lycan、Chen、Enchantress、支配头盔等会让 `npc_dota_neutral_*` 变成英雄控制单位。
   - 必须用 `team = 4` 和 `ownerClass` 过滤。
   - 如果 `sourcename` 或 `targetsourcename` 指向英雄，也需要排除。

2. 普通打野误判
   - 英雄打野怪不算拉野。
   - 必须出现线兵和真正中立野怪交互。

3. 召唤物误判
   - 树人、狼、幻象、召唤物和野怪/线兵交战，不作为拉野核心证据。

4. 中路误判
   - 中路线兵和中立单位异常交互，不输出拉野。

5. 坐标缺失
   - 很多 `DOTA_COMBATLOG_DAMAGE` 没有 `x/y`。
   - 需要用死亡日志和 `dota_model_neutral_siege_creep` 状态表补坐标。
   - 某些比赛的状态表没有 `ehandle`，此时用同秒状态点兜底，并在 evidence 中写 `synthetic:*`。

6. 同名野怪多处存在
   - 只用 `npc_dota_neutral_*` 名字会有歧义。
   - 需要结合时间、位置和状态表匹配。

7. 单条死亡噪声
   - 单个线兵或野怪死亡不一定代表完整拉野。
   - 需要使用交互数或死亡数门槛。

8. 执行英雄不确定
   - 执行者优先看是否与该组野怪交互。
   - 附近英雄只能作为位置兜底或环境证据。

9. 断线/勾兵混淆
   - 断线/勾兵看的是线兵与敌方线兵、英雄、经验的关系。
   - 拉野看的是线兵与真正中立野怪的交互。
   - 只要出现真正中立野怪交互，优先进入拉野口径。

## 11. 一句话规则

`0-15分钟内，边路线兵与 team=4 的真正中立野怪发生直接伤害/死亡交互；若combat_logs只留下单个死亡锚点，则用锚点前45秒到后5秒的状态贴近记录补足连续交战证据；若combat_logs完全缺少线兵-野怪直接交互锚点，则用状态表全局扫描550码内连续贴近，并要求连续秒数和双方掉血门槛；按线兵阵营和线兵所属路线判断为天辉/夜魇某一路拉野；成立后优先用同侧英雄/召唤物与该组野怪的交互记录归因执行者，找不到再用800码内最近同侧英雄兜底；heroes字段输出执行者，附近英雄写入结果作为复核证据。`
