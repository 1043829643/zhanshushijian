# 战术事件口径置信度假设检验方案

## 目标

`confidence` 不再作为单条事件的人工估计字段输出到最终战术事件表。

置信度改为口径模型的质量指标：每一种事件口径都要用人工批准样本做假设检验，验证该口径是否稳定、准确、可控。最终事件表只保留事实输出列，口径置信度在评估报告中单独呈现。

## 最终事件表列

最终 Excel 主表列为：

| 列名 | 含义 |
| --- | --- |
| id | 事件序号 |
| match_id | 比赛 ID |
| labels | 事件标签 |
| time_range | 事件时间段 |
| heroes | 参与/附近英雄 |
| 结果 | 事件结果 |
| 批注 | 人工复核备注 |

## 样本定义

每个口径以“人工批准表”为金标准。

样本单位可以按口径不同分为三类：

| 样本单位 | 适用口径 | 说明 |
| --- | --- | --- |
| 事件级 | 控符、开雾、肉山击杀、资源获得、带盾阵亡、囤野/堆野 | 判断某条模型输出事件是否与人工批准事件一致 |
| 候选窗口级 | 小规模冲突、团战、守塔团、高地团、肉山团、魔晶团 | 判断时间窗、标签、参战英雄、结果是否达到匹配标准 |
| 英雄级/对象级 | 对线分路、带盾阵亡、资源获得 | 判断每个英雄或资源对象是否输出正确 |

## 匹配标准

模型事件与人工事件匹配时，至少需要满足：

| 项目 | 默认标准 |
| --- | --- |
| match_id | 完全一致 |
| labels | 至少包含目标口径标签；多标签事件允许包含附加标签 |
| time_range | 瞬时事件允许 ±5 秒；窗口事件要求时间重叠，或起止误差在口径阈值内 |
| heroes | 目标英雄集合 Jaccard 相似度达到口径阈值；关键执行英雄必须命中 |
| 结果 | 对资源、击杀、死亡等结果字段，关键事实必须一致 |

不同口径可以覆盖默认标准，但必须在口径评估表中显式记录。

## 指标

对每个口径分别计算：

| 指标 | 含义 |
| --- | --- |
| TP | 模型输出且人工批准存在的正确事件 |
| FP | 模型输出但人工批准不存在或人工批注否定的事件 |
| FN | 人工批准存在但模型漏掉的事件 |
| Precision | `TP / (TP + FP)` |
| Recall | `TP / (TP + FN)` |
| F1 | `2 * Precision * Recall / (Precision + Recall)` |
| 字段准确率 | 在匹配事件中，`time_range/heroes/结果` 等字段分别正确的比例 |
| 人工批注命中率 | 人工批注指出的问题中，模型修正后被消除的比例 |

口径置信度建议定义为加权质量分：

```text
model_confidence = 0.35 * Precision
                 + 0.35 * Recall
                 + 0.20 * 字段准确率
                 + 0.10 * 人工批注命中率
```

权重可以按业务目标调整。若更重视少误报，提高 Precision 权重；若更重视不漏事件，提高 Recall 权重。

## 假设检验框架

每个口径都用二项比例检验或置信区间检验。

### 精确率检验

问题：该口径输出的事件是否足够可靠？

```text
H0: Precision <= P0
H1: Precision > P0
```

默认门槛：

| 口径类型 | P0 |
| --- | ---: |
| 事实类事件 | 0.90 |
| 战斗类事件 | 0.80 |
| 强语义/需复核事件 | 0.70 |

若 p-value < 0.05，认为该口径精确率通过检验。

### 召回率检验

问题：该口径是否没有明显漏报？

```text
H0: Recall <= R0
H1: Recall > R0
```

默认门槛：

| 口径类型 | R0 |
| --- | ---: |
| 事实类事件 | 0.90 |
| 战斗类事件 | 0.75 |
| 强语义/需复核事件 | 0.60 |

若 p-value < 0.05，认为该口径召回率通过检验。

### 字段准确率检验

对匹配事件的字段分别检验：

```text
H0: FieldAccuracy <= A0
H1: FieldAccuracy > A0
```

默认门槛：

| 字段 | A0 |
| --- | ---: |
| time_range | 0.85 |
| heroes | 0.80 |
| 结果 | 0.85 |
| labels | 0.90 |

人工批注中明确指出的字段错误，应单独统计为失败样本。

## 人工批注使用规则

人工批准文件中的 `批注` 是高优先级监督信号。

处理规则：

1. 批注指出“人物错误”，该事件的 `heroes` 字段计为失败。
2. 批注指出“结果错误”，该事件的 `结果` 字段计为失败。
3. 批注指出“地点错误”，与地点相关的标签、符点、区域字段计为失败。
4. 批注指出“漏了某英雄/事件”，对应口径计入 FN 或字段失败。
5. 批注指出“没有参与/没有囤野”，对应模型输出计入 FP 或字段失败。
6. 若批注给出明确替换答案，以批注文本作为批准答案。

## 当前结论

在未完成假设检验前，最终战术事件表不输出 `confidence` 列。

`confidence` 只在口径评估报告中出现，用于衡量每个口径模型是否通过统计检验，而不是描述某一条事件的主观可信度。
