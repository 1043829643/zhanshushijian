# Dota2 战术事件定义清单 v1.9

## 版本说明

| 项目 | 内容 |
| --- | --- |
| 版本 | v1.9 |
| 生成日期 | 2026-06-09 |
| 来源 | 基于 v1.4 定义表、团战客观数据标准 v3、以及本轮补充规则整理 |
| 主要变更 1 | 帮控符、控智慧神符统一为控符；结果列记录英雄名+拾取/罐装/反补神符；同一颗神符只记录一次 |
| 主要变更 2 | 战斗/冲突参与英雄采用通用排除规则：距离很远且未对冲突参与英雄造成伤害或控制的英雄不计入参与 |
| 主要变更 3 | 资源获得独立记录；带盾阵亡独立标注；旧抢盾/抢奶/抢刷新并入资源获得口径 |
| 表格格式 | 所有战术事件结果表最后一列为空白批注列，且所有已用单元格必须有框线 |
| 2026-06-09 更新 | 删除原单点击杀类独立标签；相关文字改为非标签化表述“单点击杀”。 |
| v1.6 变更 1 | 单点击杀候选不再作为独立标签，统一按参与规模与时间窗重叠关系归入小规模冲突或团战。 |
| v1.6 变更 2 | 单点击杀候选若造成阵亡，结果列必须补全击杀比分与死亡英雄，例如“击杀结果：天辉 1 - 0 夜魇；死亡：05:34 祈求者”。 |
| v1.7 变更 1 | 删除旧版单点候选后的重归类增加父级窗口合并：若候选已被同一小规模冲突、团战或特殊战斗事件覆盖，不再单独输出。 |
| v1.7 变更 2 | 父级窗口可来自原生战斗事件，也可来自旧版单点候选中已形成实际阵亡、并被归入小规模冲突或团战的事件。 |
| v1.7 变更 3 | 从属合并采用时间重叠与结果独立性双条件：候选与父窗口重叠比例至少 60%，父起点最多晚于候选 2 秒，父终点最多早于候选 5 秒；无独立击杀结果直接并入，有死亡结果时仅在死亡已由父事件记录时并入。 |
| v1.7 修正 4 | 肉山击杀归属按阵营资源事件记录；CHAT_MESSAGE_ROSHAN_KILL 的 player/value 字段不作为击杀英雄依据，优先使用 combat_logs roshan_death，缺失时使用 5 秒内肉山掉落归属阵营兜底。 |
| v1.8 变更 1 | 新增“肉山击杀”标签：所有肉山击杀不再作为普通资源获得输出，统一以肉山击杀事件记录。 |
| v1.8 变更 2 | 肉山击杀结果列必须写清击杀阵营，以及同一波肉山掉落的获得英雄和资源类型，例如盾、奶酪、刷新碎片、战旗等；相邻掉落行并入肉山击杀事件。 |
| v1.8 变更 3 | 新增相邻近距战斗去重：时间重叠或间隔极短、位置相近、参战英雄高度重合且无独立击杀结果的战斗候选，合并为一个事件。 |
| v1.9 变更 1 | 继续检查时间相邻战斗事件，新增“有击杀父事件吸收无独立击杀短窗口”规则。 |
| v1.9 错误原因 | v1.8 只合并了无击杀短窗口之间的重复，漏掉了父事件已记录完整击杀/死亡、相邻短窗口仅记录 BKB/TP/短控制/短伤害的情况，导致同一战斗仍拆成两行。 |

## 事件定义

| 标签 | 处理状态 | 分类 | 定义 | 判断依据 | 标准输出 | 备注 | 批注 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 对线分路 | 保留 | A：数据库可直接生成 | 0-5 分钟内按英雄在优、中、劣三条路停留时间最多的路判定；若次要路停留超过 1 分钟，额外标记游走/次要路。 | player_intervals2 位置序列；players/match_picks_bans 英雄映射。 | heroes 输出英雄及阵营；结果输出分路角色和是否存在 >1min 次要路。 | 低风险自动生成 |  |
| 勾兵 | 保留 | B：需要人工核查 | 英雄通过拉仇恨或身体位置让兵线偏离正常行进路线，并与其他兵线或野怪接上。 | 位置序列、小兵位置/交互可辅助；最终需要录像核查。 | 英雄名 + 勾兵动作开始到兵线接上的时间段。 | 当前不强制自动确认 |  |
| 断线 | 保留 | B：需要人工核查 | 前中期通常 10 分钟前，在小兵交汇前将兵线清除或显著截断。 | combat_logs 小兵死亡、英雄位置、线路判断。 | 英雄名 + 断线动作到兵线被清完的时间段。 | 可生成弱候选，需人工确认是否真为断线。 |  |
| 拉野 | 保留 | B：需要人工核查 | 通过野怪仇恨吸引部分或整波兵线停止向对方移动，转而攻击野怪。 | combat_logs 中线上小兵与中立野怪连续伤害/死亡交互；英雄位置辅助。 | 英雄名 + 拉野开始到野怪或兵线一方全部死亡的时间段；结果可写关键交互样例。 | 候选自动生成，成功与否建议复核。 |  |
| 囤野/堆野 | 保留 | B：需要人工核查 | 在野怪刷新时间将野怪勾出刷新点，使新一波野怪生成。 | player_intervals2 的 creeps_stacked/camps_stacked 增量；必要时结合位置。 | 英雄名 + 时间段；结果列默认留空，避免制造未确认结论。 | 不输出未囤野一方。 |  |
| 控符 | 保留 | A/B：数据库生成+必要时核查 | 符点刷新或符被拾取/罐装/反补时，记录该符点附近双方英雄与最终结果。覆盖普通神符、赏金、水符、护盾、智慧神符；每颗神符只记录一次。 | match_chat_events 的 RUNE_PICKUP/RUNE_BOTTLE/RUNE_DENY；player_intervals2 位置。同英雄同神符罐装后 90 秒内的拾取日志视为同一颗神符的后续激活，不另记事件。 | heroes=某符点附近双方英雄；结果=英雄名+拾取/罐装/反补某神符；无结果事件则留空。 | 替代 v1.4 的帮控符和控智慧神符。 |  |
| 蹲人 | 保留 | B：需要人工核查 | 英雄在被蹲目标 500-1000 码范围内长时间隐藏或等待，通常在树林、阴影或视野盲区。 | 英雄位置、视野、目标路径可辅助；语义强依赖录像。 | 被蹲英雄 + 蹲人英雄 + 时间段。 | 当前建议人工核查。 |  |
| 开雾 | 保留 | B：需要人工核查 | 使用诡计之雾并形成进雾队伍。 | combat_logs 中 item_smoke_of_deceit 使用和 modifier_smoke_of_deceit 添加/移除。 | 使用者 + 进入雾的英雄 + 位置；time_range 从使用到第一个雾状态消失。 | 后续进入同一波雾的英雄也计入。 |  |
| GANK | 新增 | B：需要人工核查 | N 抓 1：战斗聚类中一方至少 2 名英雄参与，另一方只有 1 名英雄参与。 | 局部伤害/控制/死亡聚类；参与英雄包括攻击者、受击者、被控制者或死亡者。 | gank 方英雄 + 被抓英雄 + 击杀/死亡变化。 | 优先于小规模冲突判定；需要校准远程消耗、路过伤害等边界。 |  |
| 小规模冲突 | 保留 | B：需要人工核查 | 介于单点击杀和团战之间的局部战斗，通常双方各 1-3 人或总人数低于完整团战，但有明确交互和结果。 | 局部伤害/控制/死亡聚类；双方参与者按通用空间/交互规则确认。 | 参与双方英雄 + 击杀/死亡变化。 | 不能把远端同步消耗并入。 |  |
| 团战 | 保留 | B：需要人工核查 | 双方多名英雄围绕同一局部目标或先手形成持续交战，通常双方各 4 人及以上或达到明确团战强度。 | 伤害、控制、死亡、买活、关键技能、位置聚类。 | 参与双方英雄 + 击杀/死亡变化 + 关键先手/伤害证据。 | 开始/结束遵循团战客观数据标准。 |  |
| 肉山团 | 保留 | B：需要人工核查 | 肉山存活时，围绕肉山坑附近约 2000 码发生的团战或资源争夺。 | 肉山存活状态、英雄位置、肉山伤害、团战聚类、ROSHAN_KILL/Aegis 事件。 | 区域 + 参与双方英雄 + 击杀/死亡变化 + 肉山结果。 | 肉山死亡后的后续追击不再误标肉山团。 |  |
| 肉山击杀 | 新增 | A：数据库可直接生成 | 肉山被击杀的资源事件。该事件按阵营归属记录，并合并同一波肉山掉落的获得结果。 | combat_logs roshan_death；match_chat_events ROSHAN_KILL、AEGIS、CHEESE、REFRESHER_SHARD、BANNER_PLANTED 等。 | labels=肉山击杀；heroes 输出获得掉落的英雄；结果输出某方击杀肉山以及各英雄获得的盾、奶酪、刷新碎片、战旗等。 | 不再误用 ROSHAN_KILL 的 player/value 字段作为击杀英雄。 |  |
| 高地团 | 保留 | B：需要人工核查 | 一方或双方在高地/基地入口/破路区域发生的团战。 | 英雄位置、高地坐标、兵营/基地/高地塔事件、战斗聚类。 | 区域 + 参与双方英雄 + 击杀/死亡变化。 | 高地前 poke 与正式开团要拆分。 |  |
| 守塔团 | 保留 | B：需要人工核查 | 在一方防御塔附近约 1000 码发生的团战或小规模守塔交战。 | tower_status_update 塔位置/血量，战斗聚类。 | 区域 + 参与双方英雄 + 击杀/死亡变化 + 防御塔上下文。 | 守塔上下文是附加标签，可与小规模冲突/团战并存。 |  |
| 肉山尝试 | 保留 | B：需要人工核查 | 对肉山造成伤害但未完成击杀或未转化为肉山团。 | combat_logs 对 Roshan 伤害；英雄位置；ROSHAN_KILL 缺失。 | 伤害英雄 + 第一下伤害到英雄离开肉山区域的时间段。 | 若双方围绕肉山开战，优先肉山团。 |  |
| 魔晶团 | 保留 | B：需要人工核查 | 折磨者/魔晶仍存活时，围绕魔晶区域发生的团战或争夺。 | miniboss damage/kill、英雄位置、团战聚类。 | 区域 + 参与双方英雄 + 击杀/死亡变化 + 魔晶归属。 | 魔晶击杀后的后续交战不再误标魔晶团。 |  |
| 资源获得 | 新增/合并 | A：数据库可直接生成 | 独立记录关键资源时间点，包括击杀肉山、击杀魔晶、获得肉山盾、奶酪、刷新碎片、战旗等。旧抢盾/抢奶/抢刷新并入本标签。 | match_chat_events: ROSHAN_KILL、MINIBOSS_KILL、AEGIS、CHEESE、REFRESHER_SHARD、BANNER_PLANTED 等。 | 获得/击杀资源的英雄或阵营 + 资源类型 + 时间点。 | 作为资源事实，不强行解释为团战。 |  |
| 带盾阵亡 | 新增 | A/B：数据库生成+必要时核查 | 携带不朽盾的英雄阵亡时独立记录；团战死亡列表中也要标注带盾阵亡。 | Aegis 获得事件、英雄死亡事件、复活/盾消耗上下文。 | 带盾英雄 + 阵亡时间 + 相关战斗/位置。 | 用于区分普通死亡和战略资源消耗。 |  |

## 全局规则

| 规则域 | 规则名称 | 定义 | 适用标签 | 输出影响 | 批注 |
| --- | --- | --- | --- | --- | --- |
| 参与归属 | 通用空间/交互排除 | 确定冲突或团战参与英雄时，英雄必须在冲突核心附近，或对冲突参与英雄造成伤害/控制，或被冲突参与英雄击杀/控制。距离很远且只在另一处发生独立交互的英雄，不计入该冲突。 | 小规模冲突、团战、肉山团、高地团、守塔团、魔晶团 | heroes 仅保留真实参与者；evidence 不使用远端独立消耗作为先手或伤害依据 |  |
| 参与归属 | 非时间限定 | 远端误识别排除不只适用于 10 分钟前，也不只适用于中路；任何时间、任何线路的同步事件都要按空间和交互关系过滤。 | 所有战斗类标签 | 减少跨地图同步伤害/控制造成的误并入 |  |
| 参与归属 | 远程技能 | 远程或全图技能若直接命中冲突参与英雄，可作为伤害/控制证据；若没有命中冲突参与英雄，只能视为另一处事件。 | 所有战斗类标签 | 可保留命中证据，但默认不把无命中的远端英雄列入参与者 |  |
| 控符 | 统一符类标签 | 普通神符、赏金神符、水符、护盾符、智慧神符等统一使用控符标签。 | 控符 | 不再输出帮控符或控智慧神符 |  |
| 控符 | 控符 heroes 字段 | heroes 记录某一符点附近的两方英雄，按符点和时间窗口附近 1000 码内英雄统计。 | 控符 | 格式为某符点附近：天辉若干；夜魇若干 |  |
| 控符 | 控符结果字段 | 结果记录英雄名+拾取/罐装/反补某神符，例如“电炎绝手罐装双倍神符”；如果符点刷新在另一侧河道导致没有结果事件，则结果留空。 | 控符 | 结果列不再塞入附近英雄名单，且必须带执行英雄名 |  |
| 控符 | 单神符单事件 | 每一颗神符只记录一次事件。若数据源同时出现 RUNE_BOTTLE 和后续同英雄同神符 RUNE_PICKUP，视为同一颗已装瓶神符，只保留罐装事件。 | 控符 | 避免 4 分钟赏金、8 分钟双倍、10 分钟隐身等被重复记录 |  |
| 资源 | 资源存活约束 | 肉山团、魔晶团只在资源仍存活且围绕资源点发生争夺时标记；资源击杀后的追击或后续团战不再因靠近资源点误标资源团。 | 肉山团、魔晶团 | 减少资源团误标 |  |
| 输出 | 标准列 | 战术事件表标准列为 id、match_id、labels、confidence、time_range、heroes、结果、批注。 | 所有输出表 | 最后一列批注为空白列，供人工复核填写 |  |
| 输出 | 框线 | 所有已用单元格必须有完整框线。 | 所有输出表 | Excel 交付表统一可审阅格式 |  |
| 战斗归类 | 单点击杀候选归并 | 不再输出单独的单点击杀类标签。候选事件若与既有团战时间窗重叠则归入团战；若与既有小规模冲突时间窗重叠则归入小规模冲突；否则按双方参与人数判定，达到团战规模归入团战，其余归入小规模冲突。 | 小规模冲突、团战 | 保留事件行与原时间窗，labels 改写为小规模冲突或团战。 |  |
| 结果字段 | 阵亡结果补全 | 单点击杀候选归并为小规模冲突/团战后，如原始事件记录了被击杀英雄，则结果列必须写入击杀比分与死亡英雄；击杀方按死亡英雄所属阵营的对方计算。 | 小规模冲突、团战 | 避免出现有英雄阵亡但结果仍为“未记录英雄击杀”的情况。 |  |
| 战斗归类 | 从属候选合并 | 删除旧版单点候选字段后，候选若与已有父级战斗窗口高度重叠，且没有独立击杀结果，或其死亡结果已经被父事件记录，则该候选不再输出独立事件，直接并入父级战斗事件。 | 小规模冲突、团战、肉山团、高地团、守塔团、魔晶团 | 避免父级团战/冲突已覆盖后，又输出多个子级集火、逃脱或短窗口战斗事件。 |  |
| 战斗归类 | 父级窗口来源 | 父级战斗窗口既包括原生识别出的战斗类事件，也包括旧版单点候选中已经造成阵亡、并按 v1.6 规则归入小规模冲突或团战的事件。 | 小规模冲突、团战 | 同一战斗片段中，一个死亡事件可吸收周边无独立击杀结果的短候选窗口。 |  |
| 战斗归类 | 从属窗口判定阈值 | 候选事件与父级战斗窗口重叠比例至少 60%；父级开始时间最多晚于候选开始 2 秒；父级结束时间最多早于候选结束 5 秒。满足该空间/时间关系后，再按结果独立性决定是否合并。 | 小规模冲突、团战、肉山团、高地团、守塔团、魔晶团 | 覆盖黑皇杖、TP 等逃脱动作导致候选窗口比团战结果窗口略长的情况。 |  |
| 资源 | 肉山击杀归属 | CHAT_MESSAGE_ROSHAN_KILL 的 player1/value 字段不视为击杀英雄。肉山击杀统一记录为阵营资源事件；归属优先取同秒 combat_logs roshan_death 的击杀阵营，若缺失，则取 5 秒内 Aegis/奶酪/刷新碎片等肉山掉落的归属阵营作为兜底。 | 资源获得、肉山团 | 结果列写作“某阵营击杀肉山”，避免误写为某个无关英雄击杀肉山。 |  |
| 资源 | 肉山击杀独立标签 | 肉山击杀从资源获得中拆出，统一记录为“肉山击杀”。同一波肉山掉落在击杀后短时间内出现时，合并入该肉山击杀事件，不再单独输出为资源获得。 | 肉山击杀、资源获得 | 结果列必须包括击杀阵营及掉落获得者，例如“夜魇击杀肉山；风暴之灵获得肉山盾”。 |  |
| 战斗归类 | 相邻近距战斗去重 | 两个战斗候选若时间重叠或间隔不超过 3 秒、位置距离不超过约 1800 码、参战英雄 Jaccard 重合度至少 50% 或重合英雄不少于 2 人，且双方均无独立击杀结果，则视为同一战斗事件并合并。 | 小规模冲突、团战、肉山团、高地团、守塔团、魔晶团 | 合并后的 time_range 覆盖全部候选窗口，heroes 取双方英雄并集，结果记为未记录英雄击杀/死亡无。 |  |
| 战斗归类 | 有击杀父事件吸收短窗口 | 若一个战斗事件已经记录击杀/死亡，另一个相邻或重叠短窗口无独立击杀结果，且二者位置距离不超过约 2500 码、共享英雄不少于 2 人或英雄重合度达到 33%，则短窗口并入有击杀父事件。 | 小规模冲突、团战、肉山团、高地团、守塔团、魔晶团 | 保留父事件击杀结果；time_range 覆盖父子窗口；heroes 取双方英雄并集；删除无独立击杀短窗口独立行。 |  |
| 质量控制 | 相邻战斗复查顺序 | 生成表格后需按战斗时间序列复查相邻 10 秒内的战斗事件。若出现共享英雄、同区域、同一击杀链或逃脱动作与父事件重叠，应优先判断是否为同一战斗被重复识别。 | 所有战斗类标签 | 防止 BKB、TP、短控制、短伤害等微事件被单独保留为战术事件。 |  |

## 输出格式

| 列名 | 含义 | 是否必填 | 备注 | 批注 |
| --- | --- | --- | --- | --- |
| id | 事件序号 | 是 | 按时间排序后从 1 开始 |  |
| match_id | 比赛 ID | 是 | 字符串或数字均可，但同一表内保持一致 |  |
| labels | 事件标签 | 是 | 多标签用 / 分隔，例如 小规模冲突 / 守塔团 |  |
| confidence | 置信度 | 是 | 0-1；自动事实类通常较高，候选类较低 |  |
| time_range | 事件时间段 | 是 | 格式 mm:ss-mm:ss；瞬时事件可起止相同 |  |
| heroes | 参与/附近英雄 | 是 | 按标签语义输出；战斗类为真实参与英雄，控符为符点附近双方英雄 |  |
| 结果 | 事件结果 | 否 | 击杀/死亡、资源结果、控符结果等；无结果事件可留空 |  |
| 批注 | 人工复核备注 | 否 | 最后一列，默认空白 |  |
